Action section

A section of any content with a corresponding 'action button' EG Show Comments or Delete Section

Some sections have no action within a block of action sections, these can be marked for consistent styling and highlighting.

States:

  • is-selected
  • is-not-selected

Comment

A comment widget consists of a comment-main widget and a list of child comments (replies). There is a form for creating new replies.

The comment may have the states is-selected or is-not-selected where the latter one is used if a different comment is selected. If no comment is selected, no state is applied.

Listing

A Listing represents a list of elements that may be filtered and/or sorted. It is generic because it can be used with very different kinds of elements.

Apart from the list of elements the listing optionally contains a form for creating new elements. This form is then always visible.

map info

Map info is used to show action-related information in the context of a map. It is different from map-info-persistant in that it should only be used for information that is related to a user action and that is not shown all the time.

Some error with a link.

persistant map info

Persistant map info is used to show general information in the context of a map. It is different from map-info in that it should be show all the time rather than in response to user action.

Navigation Bar

A bar with multpile items. The items are aligned either to the left (navbar-left) or to the right (navbar-right).

Note that the items thet are aligned to the right will be sorted starting from the right which might be the opposite of what you expect.

If there is not enough space to display all items, the leftmost of the right aligned ones will wrap.
